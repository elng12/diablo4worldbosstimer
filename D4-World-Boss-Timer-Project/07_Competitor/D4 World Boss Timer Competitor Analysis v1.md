# D4 World Boss Timer Competitor Analysis v1

## 1. 竞品对象
主要竞品：
**Helltides World Boss 页面**

竞品定位：
综合型 Diablo 4 事件工具站，覆盖 Helltide、World Boss、Schedule、Discord Bot 等事件相关功能。

## 2. 核心判断
Helltides 已经不是普通攻略页，而是成熟的 Diablo 4 工具站。它已经覆盖 World Boss Timer 的大量基础能力。

因此，我们不能只做一个相似页面。正确策略是：

**功能可以对标，页面和体验不能照搬。**

长期目标可以是比 Helltides 更全、更细，但执行顺序必须是：

1. P0 先单点突破
2. P1 功能对齐
3. P2 做深做全

## 3. Helltides 的优势
### 3.1 工具属性强
它不是纯文章，而是工具页面。用户可以快速查看事件状态、下一场 World Boss、Upcoming Events 和地图。

### 3.2 站内生态强
它不是单独的 World Boss 页面，而是 Diablo 4 事件工具生态的一部分。

### 3.3 提醒功能已有
它已经覆盖浏览器提醒、提前提醒等基础能力。因此“提醒功能”不能作为我们的唯一差异化。

### 3.4 SEO 覆盖不错
页面标题同时覆盖 timer、tracker、schedule 等关键词，FAQ 也能承接部分长尾问题。

### 3.5 用户信任基础强
作为已有工具站，Helltides 可能已经形成用户习惯和复访基础。

## 4. Helltides 的不足
### 4.1 首屏信息分散
它是综合事件站，首屏可能同时承载 Helltide、World Boss 和 Upcoming Events。对只搜索 World Boss Timer 的用户来说，信息不够聚焦。

### 4.2 World Boss 专用体验可以更强
它覆盖 World Boss，但并不是只为 World Boss 设计。我们可以在单点体验上做得更快、更清楚。

### 4.3 地点体验可细化
地图有价值，但用户实际需要的是：
- 当前地点
- 最近传送点
- 路线提示
- 区域说明
- 地点卡片

### 4.4 准确性信任表达不够突出
Timer 工具最怕时间错。我们可以把 Last Updated、Confidence、Predicted / Confirmed、Report Error 做成常驻模块。

### 4.5 长尾 SEO 可继续深挖
可以围绕以下方向做更细：
- boss-specific rewards
- exact locations
- nearest waypoints
- reminder options
- local timezone
- schedule explanation
- seasonal boss events

## 5. 竞品对比
| 维度 | Helltides | 我们 |
|---|---|---|
| 页面定位 | 综合 Diablo 4 事件工具站 | World Boss 专用工具页，后续扩展为事件工具集 |
| 首屏重点 | 多事件信息并列 | P0 只突出下一场 World Boss |
| 移动端体验 | 信息较多 | 极简倒计时 + 地点 + 提醒 |
| 地点体验 | 地图为主 | 地点 + 最近传送点 + 路线 + 延迟地图 |
| 准确性信任 | 有说明但不是核心视觉 | Last Updated + Confidence + Report Error 常驻 |
| 功能覆盖 | timer、schedule、map、reminder、FAQ | P1 对齐基础功能，P2 做得更全更深 |
| SEO 切入 | 主词强 | 长尾 FAQ + locations/schedule 专用页 + boss-specific 内容 |

## 6. 功能对标与超越路线
### P0：单点突破
目标：先比 Helltides 更快、更聚焦。

功能：
- Next Boss
- Countdown
- Local Time
- Location
- Nearest Waypoint
- Reminder CTA
- Accuracy
- Report Error

### P1：功能对齐
目标：补齐用户已经被竞品教育过的基础能力。

功能：
- Upcoming Schedule
- Locations
- Map
- Browser Reminder
- Calendar / iCal
- FAQ
- Rewards

### P2：超越竞品
目标：在细节、长尾和留存上做得更全更深。

功能：
- Waypoint route notes
- Boss-specific rewards
- Seasonal Boss
- PWA
- Discord / Telegram Alerts
- Helltide Timer
- Legion Event Timer
- Full Diablo 4 Event Schedule

## 7. 我们应该借鉴什么
可以借鉴被市场验证的功能需求：
- 倒计时
- 下一场 Boss
- Upcoming Schedule
- 地点 / 地图
- 提醒
- FAQ
- 掉落说明
- 站内事件生态

这些是用户需求，不是竞品专属。

## 8. 我们不能照搬什么
不能照搬：
- 页面布局
- 视觉风格
- 文案表达
- 地图素材
- 图片素材
- 数据展示方式
- 交互路径
- 信息优先级

否则会变成低配复刻版，用户没有理由迁移。

## 9. 核心差异化方向
### 9.1 更快
移动端打开后立即看到下一场 Boss，不让地图、广告或多事件信息抢首屏。

### 9.2 更聚焦
P0 只围绕 World Boss，不一开始扩展 Helltide、Legion、Pit 等复杂模块。

### 9.3 地点更清楚
不仅给地图，还给最近传送点、路线提示、地点卡片和区域说明。

### 9.4 准确性更透明
常驻显示：
- Last Updated
- Confidence
- Predicted / Confirmed
- Timezone
- Report Error

### 9.5 长尾更细
围绕 locations、schedule、reminder、boss-specific rewards、seasonal events 建立更完整的内容体系。

## 10. 竞争策略结论
我们的策略不是“不做 Helltides 有的功能”，而是：

**基础功能对标竞品，关键体验做出差异，后续再做得更全更细。**

最重要的是执行顺序：

1. 先用 World Boss Timer 单点赢速度和清晰度。
2. 再补齐 schedule、locations、reminder、FAQ、rewards。
3. 最后扩展成完整 Diablo 4 事件工具集。
